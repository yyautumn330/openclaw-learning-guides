## 3. 技术实现

### 3.1 架构设计

```
UI 层
├── Index.ets (首页)
├── Statistics.ets (统计)
├── History.ets (历史)
├── Metronome.ets (节拍器)
└── Profile.ets (我的)

服务层
├── RunTrackerService (主服务)
├── LocationService (定位)
├── MetronomeService (节拍器)
├── TrajectoryStore (存储)
└── GPXService (导出)

模型层
└── TrajectoryModel (数据模型)
```

### 3.2 关键技术

#### 音频播放
- 问题：SoundPool/AVPlayer API 不稳定
- 方案：AudioRenderer + 正弦波生成
- 经验：HarmonyOS 音频 API 需要多方案尝试

#### 地图集成
- 问题：动态地图 SDK 配置复杂
- 方案：高德静态地图 API
- 经验：静态地图足够且更稳定

#### 模拟测试
- 问题：跑步功能测试困难
- 方案：SimulatedTrajectoryService
- 经验：模拟功能提高开发效率

### 3.3 避坑指南

1. **SDK 路径**：`DEVECO_SDK_HOME` 不要加 `/default`
2. **音频 API**：优先用 AudioRenderer，避免 SoundPool
3. **类型系统**：ArkTS 严格类型，避免 any/unknown
4. **地图方案**：静态 API > 动态 SDK（简单场景）
