## ⚠️ 需要改进的问题

### 🔴 严重问题 (必须修复)

#### 1. 性能问题 - 内存泄漏风险
**位置**: `Index.ets` - `startUITimer()`  
**问题**: `setInterval` 创建的定时器在页面销毁时可能未正确清理  
**影响**: 长时间使用可能导致内存泄漏  
**建议**: 
```typescript
// 当前代码
startUITimer() {
  if (this.timerId !== -1) {
    clearInterval(this.timerId);
  }
  this.timerId = setInterval(() => { ... }, 1000);
}

// 建议改进
startUITimer() {
  this.stopUITimer(); // 确保先清理旧定时器
  this.timerId = setInterval(() => { ... }, 1000);
}
```

#### 2. 测试覆盖不足
**问题**: 项目没有任何单元测试或集成测试  
**影响**: 代码变更无法自动验证，回归 bug 风险高  
**建议**: 
- 添加 `PomodoroModel` 单元测试（状态管理、持久化）
- 添加 `BackgroundTaskService` 测试（计时器逻辑）
- 添加关键 UI 组件测试（设置保存、统计显示）

#### 3. 后台任务 API 使用不完整
**位置**: `BackgroundTaskService.ts` - `startBackgroundTask()`  
**问题**: 后台任务启动逻辑被简化为模拟实现  
**影响**: 应用进入后台后计时器可能停止  
**建议**: 实现完整的后台任务 API 调用

#### 4. 类型安全问题
**位置**: `BackgroundTaskService.ts`  
**问题**: `const intervalId = setInterval(...)` 后使用 `as any` 转换  
**建议**: 使用正确的类型声明
```typescript
private timerId: NodeJS.Timeout | number = -1;
```

#### 5. 成就系统时间判断 Bug
**位置**: `PomodoroModel.ts` - `checkAchievements()`  
**问题**: "早起鸟儿"和"夜猫子"成就使用当前时间判断，而非完成番茄的时间  
**影响**: 成就解锁条件不准确  
**建议**: 记录番茄完成时的时间戳

#### 6. 每日统计内存泄漏
**位置**: `PomodoroModel.ts` - `dailyStats: Map<string, DailyStats>`  
**问题**: `dailyStats` 只保存在内存中，虽然会保存到 preferences，但 Map 会无限增长  
**影响**: 长期使用后内存占用增加  
**建议**: 限制 Map 大小为最近 30 天

### 🟡 警告问题 (建议修复)

#### 1. 代码规范
- **变量命名**: `bgColor`, `primaryClr` 等缩写不一致，建议统一为 `backgroundColor`, `primaryColor`
- **魔法数字**: UI 代码中存在大量魔法数字（如 `fontSize(72)`, `width(140)`），建议提取为常量

#### 2. 状态管理
- **状态同步**: `Index.ets` 和 `Settings.ets` 之间的设置同步依赖 `reloadSettings()`，存在竞态条件风险
- **建议**: 使用事件总线或观察者模式实现跨组件状态同步

#### 3. 错误处理
- **用户提示不足**: 错误只记录到 console，用户看不到任何提示
- **建议**: 添加 Toast 或 Dialog 提示用户

#### 4. 性能优化
- **频繁刷新**: `updateTimeDisplay()` 每秒调用，可能引起不必要的 UI 重绘
- **建议**: 使用 ArkUI 的 `@Watch` 装饰器监听状态变化

#### 5. 可维护性
- **组件过大**: `Index.ets` (450+ 行), `PomodoroModel.ts` (600+ 行) 难以维护
- **建议**: 拆分为更小的组件和工具函数
