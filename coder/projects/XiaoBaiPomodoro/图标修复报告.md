# 图标颜色修复报告

## 问题描述
用户反馈应用图标显示纯绿色，不正确。设计稿图标应为番茄红 (#FF6B6B)。

## 问题原因
1. **配置缺失**: `app.json5` 中配置使用 `$media:app_icon`，但资源目录只有 `icon.png`
2. **构建缓存**: 旧版本 HAP 包可能包含错误的图标资源

## 修复步骤

### 1. 复制设计稿图标到资源目录
```bash
# 复制设计稿图标到两个位置
cp design/app-icon-512.png entry/src/main/resources/base/media/app_icon.png
cp design/app-icon-512.png entry/src/main/resources/base/media/icon.png
```

### 2. 修复代码兼容性问题
修复 `Index.ets` 中的 API 兼容性问题：
- 移除已废弃的 `.barColor()` 属性
- 使用标准的 `.backgroundColor()` 属性

### 3. 重新构建 HAP 包
```bash
export JAVA_HOME="/Applications/DevEco-Studio.app/Contents/jbr/Contents/Home"
export DEVECO_SDK_HOME="/Applications/DevEco-Studio.app/Contents/sdk"
hvigor.js clean --no-daemon
hvigor.js assembleHap --no-daemon
```

## 用户操作建议

1. **卸载旧版本**: 在设备上卸载现有的小白番茄专注钟应用
