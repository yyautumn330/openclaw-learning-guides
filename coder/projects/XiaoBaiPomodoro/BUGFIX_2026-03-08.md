# Bug 修复记录 - 2026-03-08

## 修复的 Bug

### Bug 1: 首次开始专注钟，从 25 分钟开始倒计时而不是设置的时长

**现象**: 第一次打开应用，点击开始按钮，倒计时从 25 分钟开始，而不是用户在设置中设置的时长（如 10 分钟）

**根本原因**: 
在 `Index.ets` 的 `aboutToAppear()` 中，初始化顺序错误：
- `initModel()` 是异步方法，需要加载 Preferences 中的设置
- 但 `aboutToAppear()` 没有 `await` 它，导致后续的 `updateTimeDisplay()` 在设置加载完成前就执行
- 此时 `pomodoroDuration` 还是默认值 25，所以显示 25:00

**修复方案**:
将 `Index.ets` 的 `aboutToAppear()` 改为 `async` 方法，并 `await this.initModel()`，确保在更新时间显示前设置已加载完成。

**修改文件**: `entry/src/main/ets/pages/Index.ets`

**修改内容**:
```typescript
// 修改前
aboutToAppear() {
  this.initModel();
  this.updateTimeDisplay();  // ❌ 此时 initModel 的异步操作可能未完成
  ...
}

// 修改后
async aboutToAppear() {
  await this.initModel();  // ✅ 等待初始化完成
  this.updateTimeDisplay();  // 此时设置已加载
  ...
}
```

### Bug 2: 今日完成次数和统计值不一致，重启后会从 0 开始

**现象**: 
- 统计页面显示的今日完成次数与实际完成的不一致
- 重启应用后，今日完成次数从 0 开始，没有保持

**根本原因**:
在 `PomodoroModel.ts` 的 `loadFromPreferences()` 中：
1. 加载 `todayCompleted` 和 `lastUpdateDate` 从 Preferences
2. 调用 `checkAndUpdateDate()` 检查日期是否变化
3. 如果日期变化，重置 `todayCompleted` 为 0，更新 `lastUpdateDate` 为当前日期
4. **但没有立即保存**！如果应用此时崩溃，下次启动时会从 Preferences 加载旧的 `todayCompleted` 值

**修复方案**:
在 `loadFromPreferences()` 中，调用 `checkAndUpdateDate()` 后，如果检测到日期变化导致 `todayCompleted` 重置，立即调用 `saveToPreferences()` 保存。

**修改文件**: `entry/src/main/ets/utils/PomodoroModel.ts`

**修改内容**:
```typescript
// 在 loadFromPreferences() 中
// Bug 2 修复：检查并更新日期（只在日期真正变化时重置 todayCompleted）
const dateChanged = (this.lastUpdateDate !== new Date().toLocaleDateString('zh-CN'));
this.checkAndUpdateDate();

// 如果日期变化导致 todayCompleted 重置，立即保存以确保持久化
if (dateChanged) {
  await this.saveToPreferences();
  console.info('PomodoroModel loadFromPreferences: date changed, saved reset todayCompleted');
}
```

**增强日志**:
- 在 `checkAndUpdateDate()` 中添加详细日志，追踪日期检查和重置过程
- 在 `saveToPreferences()` 中添加 `lastUpdateDate` 的日志
- 在 `incrementCompleted()` 中添加 `lastUpdateDate` 的日志
- 在 `loadFromPreferences()` 中添加详细日志
