- [ ] 所有功能正常工作
- [ ] UI 在不同设备上显示正常
- [ ] 最高分保存功能正常
- [ ] 暂停/继续功能正常
- [ ] 游戏难度递增正常

### 应用元数据
- [ ] App 名称：贪吃蛇
- [ ] 副标题：经典休闲游戏
- [ ] 描述完整准确
- [ ] 关键词已填写（最多 100 字符）
- [ ] 支持 URL 有效
- [ ] 隐私政策 URL 有效

### 截图与预览
- [ ] 6.7" 截图 (1290 x 2796)

- [ ] 状态栏：隐藏

### 定价与销售

### 隐私与合规
- [ ] 隐私政策已提供
- [ ] App 隐私标签已填写
- [ ] 内容分级：4+
- [ ] 出口合规：已回答

### 测试
- [ ] 在真机上测试
- [ ] 在不同 iOS 版本测试
- [ ] 在不同设备尺寸测试
- [ ] TestFlight 内部测试通过
- [ ] 无已知严重 bug

## 📤 提交流程

1. **Archive**
   ```bash
   # 在 Xcode 中
   # Product → Archive
   ```

2. **上传**
   - 选择 "Distribute App"
   - 选择 "App Store Connect"
   - 选择 "Upload"
   - 完成上传

3. **App Store Connect 配置**
   - 选择构建版本
   - 填写版本信息
   - 回答审核问题
   - 提交审核

4. **等待审核**
   - 通常 24-48 小时
   - 关注邮件通知
   - 准备应对可能的拒绝

## ⚠️ 常见拒绝原因

### 4.3 重复应用
- 确保应用有独特性
- 不要简单复制模板

### 2.1 应用完整性
- 应用不能是测试版
- 不能有崩溃
- 功能必须完整

### 5.1.1 隐私
- 必须提供隐私政策
- 必须填写 App 隐私标签
- 数据收集要说明

### 4.2 最低功能要求
- 应用不能太简单
- 要有足够的内容
- 用户体验要好

## 🎯 优化建议

### 提高审核通过率
1. 仔细填写所有元数据
2. 提供测试账号（如需要）
3. 在审核备注中说明特殊情况
4. 确保应用稳定无崩溃

### 提高下载量
1. 优化应用图标
2. 使用吸引人的截图
3. 写好应用描述
4. 选择合适的关键字
5. 考虑限时免费促销

### 用户评价
1. 在游戏内适当时机请求评价
2. 及时回复用户评论
3. 根据反馈持续改进
