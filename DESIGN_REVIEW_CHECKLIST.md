## 二、布局与间距检查

### 2.1 8px 网格系统

- [ ] 所有间距是 8px 的倍数（4px 用于微间距）
- [ ] 组件尺寸符合 8px 网格
- [ ] 页面边距使用标准值

| 间距类型 | 标准值 | 使用场景 | 检查 |
|---------|-------|---------|------|
| 微间距 | 4px | 图标与文字间距 | [ ] |
| 小间距 | 8px | 列表项内元素 | [ ] |
| 中间距 | 16px | 卡片内边距 | [ ] |
| 大间距 | 24px | 模块间距 | [ ] |
| 超大间距 | 32px+ | 页面分区 | [ ] |

### 2.2 页面布局

- [ ] 页面边距：16px（手机）/ 24px（平板）
- [ ] 卡片圆角：8px 或 12px
- [ ] 卡片内边距：16px
- [ ] 列表项高度：≥56px

```typescript
// ✅ 符合规范的布局
Column() {
  Text('标题')
    .fontSize(20)
    .fontWeight(FontWeight.Medium)
    .margin({ bottom: 16 })  // 2 * 8px
  
  Text('内容描述')
    .fontSize(16)
    .lineHeight(24)  // 1.5 倍行高
    .margin({ bottom: 24 })  // 3 * 8px
  
  Button('主要操作')
    .width('100%')
    .height(48)  // 6 * 8px
    .margin({ top: 16 })
}
.padding(16)  // 页面边距
```

### 2.3 响应式布局

- [ ] 使用 GridRow/GridCol 实现响应式
- [ ] 断点设置合理（sm: 320, md: 600, lg: 840）

| 设备类型 | 断点 | 列数 | 检查 |

## 六、加载与错误状态检查

### 6.1 加载状态

| 场景 | 加载方式 | 已实现 | 检查 |
|-----|---------|-------|------|
| 首次加载 | 骨架屏/Skeleton | [ ] | [ ] |

| 下拉刷新 | 刷新动画 | [ ] | [ ] |
| 上拉加载 | 加载更多指示器 | [ ] | [ ] |
| 图片加载 | 占位图 + 渐显 | [ ] | [ ] |

- [ ] 加载时间超过 2s 有进度提示
- [ ] 加载动画流畅（60fps）
- [ ] 加载失败可重试

### 6.2 错误状态

- [ ] 网络错误有明确提示
- [ ] 空状态有插图和引导
- [ ] 表单错误定位到具体字段
- [ ] 错误信息包含解决建议

```typescript
// ✅ 完整的状态处理
Column() {
  if (this.isLoading) {
    // 加载状态
    LoadingProgress()
      .width(48)
      .height(48)
  } else if (this.error) {
    // 错误状态
    Column() {
      Image($r('app.media.error_illustration'))
        .width(120)
        .height(120)
      Text('加载失败')
        .fontSize(18)
        .fontWeight(FontWeight.Medium)
      Text(this.error.message)
        .fontSize(14)
        .fontColor('#666666')
      Button('重试')
        .onClick(() => this.retry())
    }
  } else if (this.items.length === 0) {
    // 空状态
    EmptyState({
      title: '暂无内容',
      description: '点击右下角按钮添加新内容',
      action: '去添加'
    })
  } else {
    // 正常内容
    ForEach(this.items, (item) => ItemCard({ item }))
  }
}
```
