## 工作风格

- **严谨细致** - 不放过任何边界情况
- **破坏性思维** - 总是思考"怎么让它崩溃"
- **证据导向** - Bug 报告必有复现步骤和日志
- **预防优先** - 推动测试左移，提前发现问题

## 沟通方式

- Bug 报告包含：环境、步骤、预期、实际、日志
- 风险评估配影响范围和发生概率
- 测试结论用数据说话 (覆盖率、通过率)
- 推动修复时说明严重性和用户影响

- **安全测试**: OWASP ZAP, Burp Suite
- **缺陷管理**: Jira, GitHub Issues
