# ⚙️ 配置目录

> **用途**: 存放全局配置文件、日志、临时文件等

---

## 📁 目录结构

```
config/
├── README.md             # 本说明文档
├── logs/                 # 日志文件
├── tmp/                  # 临时文件
├── skills/               # 技能配置
├── agents-archive/       # 旧身份配置（归档）
└── [其他配置文件]
```

---

## 📂 各目录用途

### logs/ - 日志文件

系统运行日志：

- 会话日志
- 错误日志
- 操作记录
- 性能监控

### tmp/ - 临时文件

临时存储文件：

- 缓存文件
- 中间产物
- 测试文件
- 待处理文件

> ⚠️ **注意**: tmp 目录会定期清理，不要存放重要文件

### skills/ - 技能配置

技能相关文件：

- 已安装技能
- 技能配置
- 技能依赖

### agents-archive/ - 历史配置

旧版本身份配置归档：

- 废弃的身份文件
- 历史版本备份
- 参考文档

---

## 🗑️ 清理策略

| 目录 | 清理频率 | 保留策略 |
|------|---------|---------|
| `logs/` | 每月 | 保留最近 30 天 |
| `tmp/` | 每周 | 保留最近 7 天 |
| `skills/` | 手动 | 永久保留 |
| `agents-archive/` | 手动 | 永久保留 |

---

## 📝 配置文件

### 全局配置

- `.gitignore` - Git 忽略规则
- `BOOTSTRAP.md` - 启动配置（如需要）
- 其他全局设置

### 会话配置

- `HEARTBEAT.md` - 心跳任务（位于根目录）
- `AGENTS.md` - Agent 总控（位于根目录）

---

## ⚠️ 注意事项

1. **不要删除** `logs/` 和 `skills/` 目录
2. **定期清理** `tmp/` 目录
3. **重要文件** 不要放在 config 目录
4. **归档文件** 添加日期标记
