```

├── Week 2: 小白快跑 🟡

└── Week 4: TBD
```

**经验**:
- 小步快跑，每周发布
- 复用代码和组件
- 及时文档化

### 2. 代码组织

```
workspace/
├── coder/           # 开发者工作区
│   ├── projects/    # 项目代码
│   ├── snippets/    # 代码片段
│   └── scripts/     # 工具脚本
├── creator/         # 创作者工作区
│   ├── blog/        # 博客文章
│   └── guides/      # 教程指南
└── memory/          # 记忆文件
    └── YYYY-MM-DD.md
```

**经验**:
- 分离 coders/creator 工作区
- 每日记忆文件记录
- 定期整理归档

### 3. 备份策略

```
每日：代码提交到 GitHub (~150MB)
每周：完整备份到本地 (~140MB)
每月：清理旧备份 (保留 4 周)
```

**排除项**:
- `*.safetensors` (AI 模型)
- `venv/` (虚拟环境)
- `node_modules/` (依赖包)
- `build/` (构建输出)
