## 📚 OpenClaw 学习方法论

### 🎯 核心原则

```
✅ 用中学 (Learning by Doing)
✅ 问题驱动 (Problem-Driven)
✅ 小步快跑 (Small Steps)
✅ 持续迭代 (Continuous Iteration)
```

**不要**:
```
❌ 先学完所有文档再动手
❌ 试图一次性掌握所有功能
❌ 追求完美配置
❌ 和别人比较进度
```

- ✅ 接入一个聊天平台 (微信/飞书)

**实践项目**:
- 配置个人 AI 助手
- 安装天气技能

## 💡 学习建议

### ✅ 应该做的

1. **从实际问题出发**
   - 不要为了学而学
   - 先找到你想解决的问题
   - 再学习对应的功能

2. **记录学习过程**
   - 用 OpenClaw 记录学习笔记
   - 建立个人配置文档
   - 记录遇到的问题和解决方案

3. **加入社区**
   - Discord 提问和交流
   - 关注官方更新
   - 学习他人的最佳实践

4. **定期回顾**
   - 每周回顾学到了什么
   - 优化配置和工作流
   - 清理不用的技能

5. **分享经验**
   - 写博客分享经验
   - 在社区回答问题
   - 贡献文档和技能

---

### ❌ 应该避免的

1. **不要追求完美**
   - 先跑起来，再优化
   - 接受不完美
   - 迭代改进

2. **不要一次性学太多**
   - 每周聚焦 1-2 个技能
   - 学透再用
   - 贪多嚼不烂

3. **不要闭门造车**
   - 遇到问题先搜索
   - 在社区提问
   - 学习他人经验

4. **不要忽视安全**
   - 不要分享 API Key
   - 注意数据隐私
   - 定期更新

5. **不要放弃**
   - 遇到问题很正常
   - 坚持就是胜利
   - 社区会帮助你
